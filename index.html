    Jak se liší a kdy použít který?

    Všechny tři režimy jsou v Claude Desktop — žádná extra instalace.

    💬 Chat

    Klasický chat v Claude Desktop. Hlavní rozhraní pro práci s Power BI — přes MCP se připojuje přímo na tvůj sémantický model.

    • ✔ Propojení s Power BI přes MCP
    • ✔ Tvorba a ladění DAX měřítek
    • ✔ Ad-hoc analýzy a dotazy na data
    • ✔ Rychlá vysvětlení a konzultace
    • ⏭ Kdy použít: Vše kolem Power BI, explorační analýza, jednorázové dotazy.
    Příklad
    "Podívej se na můj datový model a navrhni chybějící metriky pro analýzu prodeje."

    💻 Claude Code

    Vyberu mu složku (lokální git repo) a Claude v ní pracuje se soubory jako vývojář — čte, píše, spouští příkazy.

    • ✔ Editace a refaktoring souborů
    • ✔ Code review a vysvětlování kódu
    • ✔ Spouštění příkazů v terminálu
    • ✔ Lokální automatizace (hooks, skripty)
    • ⏭ Kdy použít: Máš projekt nebo soubory a chceš, aby Claude sám procházel a upravoval kód.
    Příklad
    "Vytvoř DAX metriky podle souboru pozadavky.docx a po mém schválení pushni změny na git."

    🤝 Cowork

    Kolaborativní multi-agent prostředí v Claude Desktop. Umožňuje sdílet kontext s týmem, spouštět opakované tasky a orchestrovat agenty pomocí pluginů.

    • ✦ Automatizované a plánované tasky
    • ✦ Scheduled triggers (spouštění dle rozvrhu)
    • ✦ Pluginy a sdílené Skills pro tým
    • ✦ Workflow s více agenty souběžně
    • ⏭ Kdy použít: Potřebuješ opakující se workflow, automatické reporty nebo sdílet agenty s kolegy.
    Příklad
    "Každý pátek vygeneruj přehled změn v datovém modelu za týden a pošli ho týmu."

    Pokročilé Tipy

    Jak fungovat s AI asistenty chytře a efektivně.

    📊

    Kontrola a fungování limitů

    Agenti mají omezenou kapacitu zpráv a kontextu. Vaše zbývající limity najdete v Claude Desktop přes Settings → Usage, případně se ukáží jako varovný pruh v chatu při jejich docházení.

    • ●
      Current session: Limity vázané na to, kolik dat, souborů a zpráv aktuálně "držíte" v jednom chatu.
    • ●
      Weekly limits: Globální omezení využití modelu v daném týdnu pro váš celý účet napříč všemi chaty.
    🧠

    Přetížení Context Window

    Nevyplatí se psát celou historii práce a úplně všechny prompty do jednoho dlouhého chatu. Velmi rychle tím přetížíte kontextové okno (Context Window).

    Jakmile je okno plné, agent začne "halucinovat", zapomínat informace z úvodu a zbytečně rychle pálí vaše limity. Je mnohem výhodnější pro každou novou analýzu či téma založit zcela nový chat.

    🛠️

    Skills

    Co to je: Předpřipravené balíčky dovedností a znalostí (de facto pluginy). Mohou to být oficiální Anthropic Skills, osobní, nebo Third party doplňky z případného "trhu".

    Příklad:

    Interní plugin specifikovaný výhradně pro Základní analýzy Logio. Zavoláte skill a agent automaticky zná a aplikuje vaši metodiku.

    Postup založení

    • 1.
      Name & Trigger: Pojmenování a příkaz ke spuštění.
    • 2.
      Goal: Jasná definice hlavního cíle.
    • 3.
      Connectors: Napojení na specifická data/nástroje.
    • 4.
      Process: Instrukce krok za krokem.
    • 5.
      Define Output: Exaktní podoba formátu výstupu.

    SubAgents vs. Agent Teams

    SubAgents patří do Claude Code, Agent Teams do Cowork — každý se hodí na jiný typ úkolu.

    👤 SubAgents

    💻 Claude Code

    Claude Code si při složitém tasku sám spawne podagenty — každý pracuje na jiné části projektu, výsledky se sloučí.

    • ✔ Focused results (soustředěné výsledky)
    • ✔ Žádný "cross-talk" (šum v komunikaci)
    • ✔ Save tokens (zásadní úspora limitů)
    • ⏭ Kdy použít: V Claude Code, když chceš paralelně zpracovat více souborů nebo částí projektu najednou.
    Příklad promptu
    "Vezmi tento DAX kód a optimalizuj ho pro lepší výkon. Odstraň FILTER a použij proměnné."

    👥 Agent Teams

    🤝 Cowork

    V Cowork si sestavíš tým pojmenovaných agentů s rolemi — každý má svou doménu a komunikují spolu napřímo.

    • ✦ Own territory: Každý má svou doménu.
    • ✦ Direct messages: Komunikují spolu napřímo.
    • ✦ Start parallel: Agenti operují souběžně.
    • ⚠ Upozornění: Týmy spotřebovávají více tokenů.
    • ⏭ Kdy použít: V Cowork, pro workflow kde různé role (Analytik, DAX Expert, Reviewer) pracují souběžně.
    Pravidla konfigurace:
    • Own specific files: Přidělte každému specifické soubory.
    • Define output: Exaktně nadefinujte požadovaný výstup.
    • Name recipient: Určete pojmenovaného příjemce.
    • Give full context: Vždy doručte plný kontext zadání.
    Příklad promptu
    "Týme: @Architekt navrhne datový model, @DAX_Expert k němu napíše metriky a @Tester ověří všechny edge cases."

    PDF komprese přes Gemini

    ⏱

    Session timing

    Začni novou session jen tehdy, kdy jsi skutečně ready pracovat — ne jen „mrknu na to". Prázdné nebo nedokončené session zbytečně pálí context a limity.

    🪨

    Caveman Method

    Místo složitého popisu problému pošli surový kód, chybovou hlášku nebo screenshot. Claude z toho vyčte víc než z jakéhokoli popisu.

    Reply in the most concise form possible. Skip pleasantries, preambles, and recaps of my question. Prefer short declarative sentences. If a tool call is needed, run it first and show only the result. Do not narrate your steps.
    🗜

    Compact Skill

    Když session narůstá, zavolej /compact. Claude shrne vše podstatné, resetuje context okno a pokračuje bez ztráty informací.

    /compact Shrnutí musí obsahovat: (1) původní cíl/problém, (2) klíčová rozhodnutí a jejich důvody, (3) kód, konfigurace nebo data, na kterých jsme se shodli (v blocích kódu), (4) otevřené otázky a další kroky. Vynech small talk. Optimalizuj shrnutí tak, aby ho jiný Claude v novém chatu mohl snadno uchopit.

    Knihovna Promptů

    Ověřené příkazy připravené k okamžitému zkopírování.

    📅

    1. Kalendářní tabulka

    Základní stavební kámen. Vytvoří komplexní kalendář propojitelný s fakty a dynamicky označí pracovní dny.

    Create a comprehensive DAX Date table. Include columns for Date, Year, Quarter, Month Name, Month Number, Week Number, and Day of Week. Add a boolean column for IsWorkingDay (assuming Monday-Friday). Before generating the code, please outline your plan for my approval.
    ⏳

    2. Time Intelligence

    Vygeneruje běžné metriky pro porovnání v čase (YTD, YoY) z vašeho existujícího základního měřítka.

    Based on the existing measure [Total Sales], generate standard time intelligence measures including: YTD, QTD, MTD, Previous Year (PY), YoY variance in absolute numbers, and YoY variance percentage. Please explain your approach before writing the DAX.
    🧹

    3. Úklid názvů

    Automatické přejmenování sloupců (odstranění podtržítek atd.) pro perfektní vizuální konzistenci v reportech.

    Analyze naming conventions across all tables and columns. Rename fields for consistency, fix casing (e.g. Title Case), and remove unnecessary prefixes. Ensure all DAX measures update accordingly. Show me the mapping of old names to new names for my approval before executing.
    ⚡

    4. Optimalizace DAX

    Agent najde bottlenecky ve výkonu složitého DAXu a navrhne mnohem rychlejší variantu výpočtu.

    Review the following DAX measure for performance bottlenecks. Suggest an optimized version using variables (VAR) and efficient filter context evaluation. Explain the performance improvements in your plan before outputting the final DAX.
[VLOŽTE SVŮJ DAX ZDE]
    ⭐

    5. Návrh datového modelu (Star Schema)

    Nechte AI navrhnout optimální Star Schema model ze surových dat. Výborné na začátku projektů, kdy agent identifikuje fakta, dimenze a klíče.

    Analyze the following list of raw flat tables and their columns. Propose an optimal Star Schema design. Identify which tables should be Fact tables and which should be Dimensions. Suggest the necessary relationships (1-to-many) and surrogate keys. Present your structural plan for my approval.
[VLOŽTE SEZNAM TABULEK/SLOUPCŮ]

    NotebookLM

    Tvůj vlastní AI výzkumný asistent postavený na tvých dokumentech.

    📚

    Co to je?

    Google nástroj, který vezme tvoje vlastní soubory a vytvoří z nich chytrého asistenta. Nahraješ zdroje, ptáš se — a dostáváš odpovědi s přesnými citacemi.

    • ✔ PDF dokumenty, Google Docs, weby, YouTube videa
    • ✔ Odpovídá pouze ze zdrojů — žádné halucinace
    • ✔ Sdílitelné notebooky pro tým
    • ✔ Export zdrojů jako Knowledge Base do Gemini Gems
    💡

    Jak to použít v praxi

    1. 1
      Vytvoř notebook na notebooklm.google.com a nahraj zdroje (metodiky, best practices, dokumentace).
    2. 2
      Ptej se přirozeně — "Jaký je postup pro X?" nebo "Shrň klíčové body z dokumentu Y."
    3. 3
      Sdílej notebook s týmem — každý se ptá bez nutnosti číst celé dokumenty.
    4. 4
      Nahraj jako Knowledge Base do Gems — zdroje z NotebookLM přidej do Gemini Gemu, aby je měl asistent vždy k dispozici.

    Gemini Gems

    Vlastní AI asistent s instrukcemi a znalostní bází — tvůj specializovaný expert.

    💎

    Co je Gem?

    Gem je přizpůsobená verze Gemini s vlastními instrukcemi a nahranými soubory jako znalostní bází. Funguje jako specializovaný asistent, který vždy "ví" co potřebuje.

    Rozdíl oproti běžnému chatu:

    • ✦ Pamatuje si instrukce — nemusíš je opakovat každý chat
    • ✦ Vlastní knowledge base — nahraješ soubory jednou
    • ✦ Lze sdílet s týmem — všichni používají stejného asistenta
    ⚙️

    Jak vytvořit Gem

    1. 1
      V Gemini klikni na Gem Manager → Nový Gem.
    2. 2
      Pojmenuj Gem — např. "Logio Data Analytik".
    3. 3
      Napiš instrukce — co má Gem dělat, jaký má styl, na co se specializovat.
    4. 4
      Nahraj soubory — metodiky, best practices, šablony, interní dokumentace.
    5. 5
      Uložit a sdílet s týmem — všichni mají přístup ke stejnému asistentovi.

    Pravidelné Prompty

    Připravené šablony k okamžitému zkopírování a naplánování.

    Pondělní AI přehled novinek

    Naplánovat jako pravidelný weekly task každé pondělí ráno.

    Každé pondělí ráno v 8:00 mi udělej souhrn AI novinek za uplynulý týden. Zaměř se na:
1. Novinky ohledně Claude (Anthropic) a Gemini (Google) — nové funkce, modely, změny
2. Zajímavé AI use cases relevantní pro datovou analytiku a Business Intelligence
3. Doporučení: jak bych mohl konkrétní novinku využít ve svých projektech

Formát: přehledný seznam s krátkými bullet pointy, max 1 stránka.

📌 Note: Gemini tento prompt automaticky nabídne jako Scheduled prompt — stačí potvrdit čas a frekvenci.

    Analýza dashboardu / grafu

    Nahraj screenshot a použij tento prompt pro rychlou analýzu.

    Analyzuj přiložený screenshot dashboardu/grafu. Popiš:
