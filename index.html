Knihovna promptů pro Claude

Zefektivněte svou práci v Power BI pomocí těchto ověřených příkazů připravených přímo pro AI asistenta Claude.

Zlatá pravidla (Best Practices)

Než svěříte Claudeovi úpravy vašeho datového modelu, vždy postupujte obezřetně:

  • Vždy žádejte o plán: Přidejte ke svému promptu instrukci, ať Claude nejprve navrhne postup. (Např.: "Nejprve mi prosím napiš krok za krokem plán toho, co přesně se chystáš udělat.")
  • Vyžadujte schválení: Zakázejte agentovi psát finální kód, dokud mu jeho plán výslovně neodsouhlasíte.
  • Testujte postupně: Plný přístup a komplexní úkoly zadávejte až po odzkoušení na menších částech modelu.

Osvědčené statické prompty

1. Vygenerování komplexní kalendářní tabulky

Create a comprehensive DAX Date table. Include columns for Date, Year, Quarter, Month Name, Month Number, Week Number, and Day of Week. Add a boolean column for IsWorkingDay (assuming Monday-Friday). Before generating the code, please outline your plan for my approval.

Základní stavební kámen pro Time Intelligence funkce. Vytvoří plně funkční kalendář propojitelný s vašimi fakty.

2. Tvorba sady základních Time Intelligence měřítek

Based on the existing measure [Total Sales], generate standard time intelligence measures including: YTD, QTD, MTD, Previous Year (PY), YoY variance in absolute numbers, and YoY variance percentage. Please explain your approach before writing the DAX.

Ušetří spoustu manuálního psaní. Rychle vygeneruje všechny obvyklé metriky pro porovnávání v čase.

3. Přejmenování polí pro konzistenci (Úklid názvů)

Analyze naming conventions across all tables and columns. Rename fields for consistency, fix casing (e.g. Title Case), and remove unnecessary prefixes. Ensure all DAX measures update accordingly. Show me the mapping of old names to new names for my approval before executing.

Velmi užitečné pro surová data (např. převod "dim_product_name" na "Product Name").

4. Optimalizace výkonu DAX měřítka

Review the following DAX measure for performance bottlenecks. Suggest an optimized version using variables (VAR) and efficient filter context evaluation (e.g., avoiding FILTER on whole tables). Explain the performance improvements in your plan before outputting the final DAX.
[INSERT YOUR DAX HERE]

Skvělé, když se vám načítají vizuály pomalu. Claude najde zbytečné iterace a navrhne rychlejší výpočet.

5. M kód: Paginace API přes Power Query

Create a Power Query M function that takes a dynamic API URL and a bearer token as parameters. It needs to handle pagination using the 'next_page' URL provided in the API JSON response, and return a combined table of all results. Outline the steps and logic for my approval before writing the M code.

Záchrana při stahování dat z REST API, které vrací výsledky po stránkách (např. po 100 záznamech).

6. Dynamické Row-Level Security (RLS)

Design a dynamic Row-Level Security (RLS) DAX expression for the 'Sales' table. Users should only see data for their assigned Region based on the 'UserSecurity' table (matching USERPRINCIPALNAME()). Account for managers who have a '*' indicating access to all regions. Provide the plan and DAX logic for approval first.

Řeší běžný bezpečnostní požadavek, kdy řadový zaměstnanec vidí jen svá data a manažer vidí vše.

7. Návrh datového modelu (Star Schema)

Analyze the following list of raw flat tables and their columns. Propose an optimal Star Schema design. Identify which tables should be Fact tables and which should be Dimensions. Suggest the necessary relationships (1-to-many) and surrogate keys. Present your structural plan for my approval.
[INSERT TABLES/COLUMNS HERE]

Zásadní prompt při startu nového reportu. Nechte Claudea promyslet strukturu (fakta vs. dimenze) ze surových dat.

8. Dynamická segmentace zákazníků

Write a DAX measure to dynamically segment customers based on their total sales into 'High', 'Medium', and 'Low' tiers using a disconnected 'Segmentation Parameters' table with Min and Max columns. The measure should count the number of customers in each tier. Explain the logic for approval before outputting code.

Pokročilý DAX vzorec, který umožní interaktivně rozdělovat zákazníky nebo produkty do skupin (např. ABC analýza).

Rychlý Setup: Krok za krokem k automatizaci

Pro propojení Claude s vašimi lokálními daty a Power BI projekty pomocí MCP si postupně projděte těchto 6 kroků. Vše naklikáte za pár minut.

1. Visual Studio Code

Stáhněte si základní editor. Bude fungovat jako most pro lokální běh serverů.

Stáhnout VS Code

2. Claude Desktop

Stáhněte si aplikaci do PC. Webová verze lokální MCP připojení nepodporuje.

Stáhnout Claude

3. MCP Rozšíření

Otevřete VS Code, jděte do Extensions (Rozšíření) a nainstalujte si MCP Server rozšíření.

4. Zkopírovat Kód

Najděte cestu k instalaci ve VS Code (např. %USERPROFILE%\.vscode\extensions). Zkopírujte JSON a doplňte vlastní jméno (pozor na dvojitá lomítka \\):

{
  "mcpServers": {
    "powerbi-modeling-mcp": {
      "command": "C:\\Users\\TvojeJmeno\\.vscode\\extensions\\analysis-services.powerbi-modeling-mcp-0.1.9-win32-x64\\server\\powerbi-modeling-mcp.exe",
      "args": ["--start"],
      "type": "stdio"
    }
  }
}

5. Nastavení Claude

V aplikaci Claude jděte do Settings > Developer a zkopírovaný kód vložte do konfiguračního souboru.

6. Restart a je hotovo!

Nejlepší rada IT oddělení: Vypněte a zapněte celý počítač! Tím se všechny služby spolehlivě propojí a už to frčí.
